# Базовые упражнения (`0x`)

## `03-qsort-str`

Разберитесь, как вызвать функцию `qsort` для сортировки массива строк, где каждая строка — это `const char*`.

Обратите внимание, что `<` для `const char*` будет сравнивать указатели, а не ASCIIZ-строки.

## `04-address-sanitizer`

Убедитесь, что программа `04-address-sanitizer.c`, скомпилированная с Address Sanitizer (`-fsanitize=address` для GCC/Clang) успешно сообщает о выходе за границу массива в строчке `8`.

Строчку `malloc: nano zone abandoned due to inability to reserve vm space`. под macOS игнорируйте.

Вывод должен начинаться примерно как в файлах `04-address-sanitizer.expected.*.txt`, у вас должен быть очень похожий. Для показа номеров строк включите отладочную информацию: `-ggdb` (или просто `-g`) для GCC/Clang. Если у вас показывает номера строк и без этого ключа — ничего страшного.

Дополнительно:
1. Обратите внимание, что после нахождения ошибки выполнение прекращается: строчка `COMPLETED` при включённом Address Sanitizer на экран не попадает.
2. Избавьтесь от выхода за пределы массива и проверьте, что будет, если запустить корректную программу с Address Sanitizer.
