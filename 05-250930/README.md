# Общее

Цель занятия: познакомиться с `malloc` и научиться пользоваться `gdb`.

# Базовые упражнения (`0x`)

## `01-malloc`
Создайте динамический массив из `n` интов. Используйте `malloc`/`free`.

Убедитесь что у вас не утекает память, запускайте вашу программу с address-sanitizer (см. упражнение с прошлой практики `04-250923/04-address-sanitizer`).

## `02-even-only`

Реализуйте функцию, принимающую массив целых чисел и его длину, и возвращающую новый массив, содержащий только чётные числа.

## `03-create-matrix`
Продолжение задания `01-malloc`: сделайте функцию `create_matrix`, которая создаёт двумерную матрицу `N`x`N` и заполняет её числами от `1` до `N*N`. Также создайте симметричную функцию `destroy_matrix` для удаления матрицы.

Есть два решения: одно использует `n + 1` вызов `malloc`, другое использует `2` вызова `malloc`. Напишите первое.

## `04-getline-realloc`

Почти реализована функция `read_line`: она читает строку произвольной длины и возвращает на неё владеющий указатель `char*`. Допишите недостающий кусок, занимающийся расширением буфера.

Сохраните линейную сложность работы алгортима. Если память на куче закончилась, то `read_line` тоже должна вернуть `NULL`. Память не должна утекать ни в коем случае: ни при обычной работе (проверьте санитайзером и valgrind), ни если функция вернула `NULL`.

Подсказка: почитайте про функцию `realloc` из стандартной библиотеки.

## `05-gdb`

Послушайте рассказ семинариста про отладчик `gdb`. У вас также есть [шпаргалка](../common/gdb.md) в папке `common`. 

## Упражнения для закрепления материала (`1x`)

## `10-valgrind-uninitialized`

Это упражнение требует инструмент Valgrind, который есть только под Linux. Если у вас не установлен Valgrind, то его нужно установить: `sudo apt install valgrind` для Ubuntu.

1. Скомпилируйте файл `10-valgrind-uninitialized.c` без address sanitizer и без предупреждений компилятора.
1. Запустите его из консоли не обычным образом, а в виртуальной машине valgrind: не `./main`, а `valgrind ./main`.
1. Убедитесь, что вывелось несколько ошибок про использование неинициализированного значения во время выполнения,
   примерно как в файле `10-valgrind-uninitialized.expected.01-before-fix.txt`.
   * Обратите внимание, что выполнение программы после ошибок продолжается.
   * Для показа номеров строк может потребоваться добавить при компиляции ключо `-ggdb`.
     Если у вас показывает номера строк и без этого ключа — ничего страшного.
1. Проинициализируйте `x`, перекомпилируйте, снова запустите.
   Ошибки должны исчезнуть, примерно как в файле `10-valgrind-uninitialized.expected.02-after-fix.txt`.
1. А теперь перекомпилируйте с address sanitizer и запустите под Valgrind.
   Программа должна даже не начать выполняться, примерно как в файле
   `10-valgrind-uninitialized.expected.03-asan-conflict.txt`.
   Причина: address sanitizer и Valgrind несовместимы друг с другом, слишком похожие вещи делают разными способами.

## `11-bubble-sort`
