# Терминология

## Репозиторий

Репозиторий -- корневая папка вашего проекта

## Коммит

Коммит -- версия репозитория, версии связаны друг с другом и идут поочередно. От коммита к коммиту можно откатываться

## Ветки

Коммиты образуют собой древовидную структуру, после одного коммита может идти два различных коммита, не связанных друг с другом. Для того чтобы такой ситуации достичь, необходимо создать новую ветку, в которой создать новый коммит. Тогда у вас появится ещё одна ветка в вашем репозитории

# Основные комманды

## git log

Показать историю коммитов текущей ветки

## git clone

Склонировать репозиторий ("скачать" его): `git clone https://gitlab.com/blonded04/hse-spb-cpp-2024-is.git` 

## git pull

Подтянуть изменения ветки с удаленного репозитория ("из интернета"). Синтаксис: `git pull <название-remote> <ветка>`, например: `git pull origin master`

## git push

Загрузить изменения ветки на удаленный репозиторий ("в интернет"). Cинтаксис: `git push <название-remote> <ветка>`, например: `git push origin hw01`

## git add

Добавить измененный файл в текущий коммит: `git add file1.ext1 file2.ext2 directory/file3.ext`

## git commit

Создать новый коммит, идущий после текущего, который изменяет файлы, добавленные в текущий коммит.

## git branch

`git branch` -- вывести список веток.

`git branch <новая-ветка>` -- создает новую ветку с именем `<новая-ветка>`

## git switch

Переключить ветку с текущей на интересующую. Синтаксис: `git switch <интересующая-ветка>`
