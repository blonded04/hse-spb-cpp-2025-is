# make

GNU Make - система сборки. Системы сборки помогают программистам автоматизировать сборку и описывать правила компиляции и компоновки программ. `make` упрощает процесс сборки, разрешая сложные зависимости файлов внутри проектов и отслеживая какие части проекта нужно перекомпилировать.
* Вместо перекомпилирования всей программы каждый раз, `make` пересобирает только файлы, которые были изменены. Это сохраняет *огромное* количество времени в больших проектах.
* `make` позволяет описывать сборку в структурированном виде, явно указывая зависимости внутри проекта для каждой из его частей.
* `make` позволяет не только описывать сборку, но и автоматизировать другие задачи: запуск тестов, очистку директорий со временными файлами, и многое другое.

# Терминология
## Makefile
`Makefile` - файл, содержащий правила и инструкции для сборки проекта. В нём описано каким образом (каким компилятором, с какими флагами, с какими библиотеками) компилировать ваши файлы и какие зависимости существуют внутри него.

## Targets (цели)
Таргеты - чаще всего исполняемые или ELF файлы, которые вы хотите собрать. `Makefile` описывает рецепт для создания каждого таргета. У каждой цели могут быть зависимости от других целей или файлов.

## Rules (правила, рецепты)
Правила в `Makefile` описывают как получить таргет: описываются зависимости (иногда называются предпосылками: другие необходимые таргеты для сборки этого таргета) этого таргета и команды, с помощью которых его можно сгенерировать. Обычно рецепт имеет такой формат:
```
target: dependencies
    command
```

## Переменные
Внутри `Makefile` можно заводить переменные.

# Пример
```
CC = gcc
CFLAGS = -Wall -Wextra -std=c17
TARGET = application

SOURCES = main.c utils.c
OBJECTS = main.elf utils.elf

$(TARGET): $(OBJECTS)
    $(CC) $(CFLAGS) -o $(TARGET) $(OBJECTS)

main.elf: main.c
    $(CC) $(CFLAGS) -c main.c -o main.elf

utils.elf: utils.c
    $(CC) $(CFLAGS) -c utils.c -o utils.elf

clean:
    rm -f $(OBJECTS) $(TARGET)

.PHONY: clean
```

Первая цель в этом `Makefile` это `$(TARGET)` (значение переменной `TARGET`, то есть `application`). `make` умеет собирать цели явно, если его об этом попросить: `make utils.elf` или `make clean` исполнит рецепты для `utils.elf` и `clean`, если для них есть что исполнять (если `utils.elf` уже был собран, то нет смысла его пересобирать). Первым рецептом в этом `Makefile` идёт рецепт для `application`, поэтому его можно собирать не указывая его названия, а просто написав `make`.

Правило для `application` описывает, что `application` зависит от значения переменной `OBJECTS` (`main.elf utils.elf`): без этих файлов нельзя скомпилировать `application`. Команда для сборки `application` вызывает `CC` (от "c compiler"), передаёт ему флаги `CFLAGS`, указывает что выходной файл должен быть значением переменной `TARGET` (application), и скомпилировать вместе содержимое переменной `OBJECTS` (`main.elf` и `utils.elf`). Правило описывает получение итогового исполняемого файла.

Правила для `main.elf` и `utils.elf` описывают сборку ELF файлов, и указывают, что пересобирать эти файлы нужно в случае изменения файлов-зависимостей: `main.c` и `utils.c` соответственно.

Цель `clean` описывает как очистить порожденные `make`'ом файлы. Запускается как `make clean`.

`.PHONY` это особая цель, которая не связана ни с каким файлом: таргеты помеченные как phony таргеты исполняются всегда, то есть при вызове `make clean` всегда будет вызван таргет `clean`: независимо от того, нужно ли пересобирать зависимости.

# Дополнительно
## Директории
`make` отслеживает изменения файлов по изменению временных меток, а у директорий временные метки меняются в случае изменения содержимого директории, что, в случае использования директорий как зависимостей, может привести к неожиданным последствиям.

## Порядковая зависимость
Order-only prerequisite позволяет указать, что определённый таргет должен существовать до того, как будет выполнен основной таргет, но не влияет на его актуальность.

Например, если у вас есть цель, которая требует наличия директории, в которой будут храниться объектные файлы, вы можете использовать order-only prerequisite для этой директории. Это означает, что если директория уже существует, она не будет пересоздана, и Make не будет считать её причиной для пересборки основной цели.

Пример:
```
obj/util.elf: src/util.c | obj
```
