# GDB
GDB (GNU Debugger) — это отладчик (дебаггер) для программ на языках C, C++, и многих других. Он позволяет анализировать выполнение программы, находить и исправлять ошибки.

# Основные команды GDB
Для работы с `gdb` программу необходимо собрать с отладочными символами (флаг `-ggdb`) — в рамках нашего курса не понадобиться пользоваться GDB на программах без отладочных символов, хотя это и возможно.

`gdb ./your-program` запускает GDB и позволяет вам писать следующие команды для GDB.

## `break`
`break S` (или `b S`) устанавливает breakpoint (точку останова) в `S`: определенной функции (если `S` - имя функции) или строке кода (если `S` имеет вид `filename.ext:linenumber`). В этой точке программа приостанавливается, и GDB позволяет анализировать программу в приостановленном состоянии - например, читать значения переменных.

## `run`
`run` (или `r`) запускает программу внутри отладчика.

## `step` и `next`
На точке останова можно продолжить исполнять программу по шагам:
* `next` (или `n`) — выполнить следующую строку (перейти к следующей строке кода).
* `step` (или `s`) — зайти внутрь вызываемой на текущей строке функции.

## `print`
`print V` (или `p V`) — вывести переменную `V`.

Дополнительно:
* `p/x addr` — вывести указатель `addr`.
* `p/s str` — вывести строку `str` как строку, а не как указатель.

## `continue`
`continue` (или `c`) — продолжить исполнение программы пока не встретится следующий breakpoint.

## `bt`
`bt` выводит текущий stacktrace.

## `display`
`display V` — включает автоматический вывод в консоль значения переменной `V` каждый раз когда GDB останавливается на breakpoint или после `step`.

## `layout src`
`layout src` — более красивый TUI (terminal user interface) для GDB.
