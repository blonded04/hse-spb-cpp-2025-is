# Общее
Цель занятия: 
1. Поурпражняться с правилом трёх и перегрузкой операторов.
2. Научиться компилировать программу с библиотекой для тестирования `doctest`.
3. познакомиться с явным и неявным конструктором, поупражняться с перегрузкой операторов.

Компилируйте два файла вместе: `doctest_main.cpp` и `task.cpp` (в нём лежат тесты).

## `01-auto-ptr`

В этом задании от вас хочется получить реализацию владеющего указателя над `int`, удовлетворяющую правилу трёх. Указатель является владеющим в том смысле, что он сам управляет своим временем жизни: сам вызывает `delete` в своём деструкторе, **не допуская при этом вызова `delete` дважды на одном и том же коде**.

Это упражнение на [правило трёх](https://en.cppreference.com/w/cpp/language/rule_of_three) языка C++. Скопируйте ваше решение задания `15-raii-int-ptr` с прошлой практики.

1. Добавьте следующие методы: 
   * `getPointer`, возвращающий указатель, которым в данный момент владеет `RaiiIntPtr`;
   * `releasePointer`, отменяющий владение указателем, которым в данный момент владеет `RaiiIntPtr`.
2. Сделайте так, чтобы класс `RaiiIntPtr` удовлетворял правилу трёх.
3. Напишите тесты к вашему коду: 
   * Протестируйте каждый метод; 
   * Протестируйте разные сценарии для каждого метода.

**Подумайте** о том, какие минусы у реализованного вами решения.

## `02-conversions`
А компилируете вы файлы `02-conversions.cpp` с тестами и `doctest_main.cpp`.

Сначала закомментируйте все подзадачи, а потом решаейте по одной.

1. Создайте класс `Foo` с двумя полями: строчка `a` и инт `b`.
   Добавьте неявный (implicit) конструктор от двух таких же аргументов,
   который инициализирует поля.
2. Добавьте ещё две перегрузки (overload) конструктора: от одного инта (в строчку записывается `from int`) и от инта со строкой.
   Чтобы сократить реализацию, используйте параметры по умолчанию вроде `void foo(int a, int b = 5)`.
3. Добавьте явный (explicit) конструктор от одной строчки, которая инициализирует поле.
   В интовое поле запишите `-1`.
   Теперь есть две перегрузки конструктора.
   Используйте делегирующий конструктор, чтобы чуть сократить реализацию.
   Поймите, почему этот пункт нельзя решить добавлением параметра по умолчанию в конструктор из первого пункта.
4. Добавьте оператор неявного (implicit) преобразования в пару из строки и инта.
   Возвращает лежащие в полях значения.

## `03-operators`
Скопируйте `02-conversions-foo.hpp` в `03-operators-foo.hpp`, __сделайте поля приватным__ и дописывайте.

1. Добавьте один оператор `==`, который сравнивает два поля у двух структур `Foo`.
2. Добавьте оператор явного (explicit) преобразования в инт.
   Возвращает лежащее в поле значение.
   Если у вас сломался оператор `==` — перепроверьте, что вы добавили именно явное преобразование.
3. Добавьте оператор `+=`, который прибавляет к полям значения из полей правого операнда.
4. Убедитесь, что `+=` возвращает неконстантную ссылку на свой левый операнд, чтобы его можно было использовать по цепочке.

## `04-operators-extra`
Скопируйте `03-operators-foo.hpp` в `04-operators-extra-foo.hpp` и дописывайте.

1. Добавьте префиксный оператор `++`, который инкрементирует инт и дописыват `1` в конец строки.
   Обратите внимание, что должно должен возвращаться тот же объект, который мы инкрементируем, причём по ссылке, чтобы работала цепочка.
2. Добавьте постфиксный оператор `++`.
3. Добавьте оператор вывода в поток `<<`.
   На точные требования FormattedOutputFunction забить, просто выведите поля через пробел.
4. Добавьте оператор чтения из потока `<<`.
   На точные требования FormattedInputFunction забить, просто считайте поля.
