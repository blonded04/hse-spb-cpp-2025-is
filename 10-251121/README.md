# Общее
Цель занятия: научиться работать побайтовым вводом-выводом trivially copyable структур.

Разрешается изменять только код, помеченный `TODO`.

# Синтаксис (`0x`)
## `01-reinterpret-cast`
Здесь создаётся trivially copyable структура `Point`.
Скопируйте её не при помощи `=`, а при помощи функции `copy_bytes` (аналог стандартной `std::memcpy`),
которая копирует байты, а не поля.

## `02-trivially-copyable`
Допишите функции `write_to` и `read_from`, которые сохраняют/читают trivially copyable структуру в/из файла.
Будьте осторожны: файл следует открывать в бинарном режиме, иначе будет несовместимость между Windows и Linux/macOS.

Должен получиться файл `02.bin`, наверняка совместимый с `02.mingw-w64-x86_64-gcc.bin`.
Байты padding могут иметь любое значение.
Поймите, как сохранились `int`'ы; это не единственный способ, но популярный (little-endian).

## `03-packed`
Похоже на предыдущее, но теперь вам надо сделать структуру `Header` упакованной, чтобы не было padding,
а выравнивание в памяти допускалось любое.

Должен получиться файл `03.bin`, наверняка совместимый с `03.mingw-w64-x86_64-gcc.bin`.

Подсказка: определяйте структуру между директивами `#pragma pack`.
```c
#pragma pack(push, 1)
// your struct goes here
#pragma pack(pop)
```

## `04-small-one`
Научитесь базово понимать формат [BMP](https://ru.wikipedia.org/wiki/BMP): где заголовки, где размеры изображения, где сами пиксели и в каком порядке,
какое используется выравнивание.
Используйте графический редактор (Paint, GIMP), а также `hexdump -C` или `xxd -g 1` (идёт в комплекте с `vim`) для анализа байт в файле.
Также можете использовать обратную команду `xxd -g 1 -r`, чтобы создать файл из произвольных байт.
