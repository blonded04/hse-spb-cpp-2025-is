# Продвинутый синтаксис (`2x`)
## `20-any`
Здесь мы попробуем написать костяк `std::any` — контейнера,
который умеет хранить один элемент любого типа `T`, который подддерживает две операции:
удаление (то есть имеет деструктор) и вывод в поток через `<<`.
При этом сам any должен быть одним конкретным типом, не шаблоном.
Например, чтобы можно было из функции `generate()` возвращать какое-то значение неизвестного типа.
`any` сильно похож на `std::function`, только набор операций другой: вывод на экран вместо вызова функтора.

Так как мы не можем использовать шаблоны, нам остаётся только другой полиморфизм: динамический, через виртуальные функции.
Так как нам всё равно надо хранить объект типа `T` и его размер неизвестен, его придётся хранить на куче.
Осталось прикрутить наследование и куда-то добавить шаблоны: как минимум конструктор any должен быть шаблонный.
