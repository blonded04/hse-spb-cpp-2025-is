# Общее
Цель занятия: научиться использовать исключения в программах из нескольких функций.

Перед выполнением упражнения обязательно прочитайте тут, как его тестировать.

Перед запуском упражнения убедитесь, что вы понимаете, почему текущее решение неверное (даже если компилируется).

Разрешается изменять только код, помеченный `TODO`.

Рекомендуется обсудить с преподавателем: `04-iostream-exceptions` (в том числе про `eofbit`), `20-mixed-exceptions`, `22-injection-attack`.

# Синтаксис (`0x`)
## `01-catch-standard`
Программа в цикле считывает два номера и выводит на экран поэлементную сумму векторов с такими номерами.
Добавьте обработку ошибки: если номер вектора слишком большой, надо вывести какое-нибудь сообщение (неважно какое) и продолжить работу.
`operator[]` по умолчанию исключений не бросает, посмотрите документацию.

Протестируйте: при запуске на `.in` файле должен получаться вывод вроде того, что в `.out`-файле (строчки 3 и 4 могут сильно отличаться, но это всё ещё должно быть сообщение об ошибке).

Теперь у вас есть отдельно короткий код с happy path и отдельно код с sad path.

## `02-throw-basic`
Измените только функцию `sum` так, чтобы при несовпадении размеров векторов выводилось только сообщение об ошибке в нужном формате,
смотри `.out`-файл.
Не создавайте свой класс, используйте стандартные.

Тестируется как и прошлое задание.

Теперь вы можете обнаруживать ошибки в `sum`, вызывать его как угодно сложно, а обрабатывать их там, где удобно.

## `03-my-exception`
Создайте свой класс `vector_size_mismatch`, чтобы можно было отличить ваше исключение от стандартного по типу,
плюс было удобнее его кидать.
Не надо вручную выделять память под `char*`, вы тогда намучаетесь с правилом пяти и/или утечками памяти.

## `04-iostream-exceptions`
Добавьте в программу A+B обработку ошибок чтения из файла: оберните решение задачи в try-block
и переведите `ifstream` в режим выбрасывания исключений при ошибках.
Ошибки у потоков [бывают разные](https://en.cppreference.com/w/cpp/io/ios_base/iostate): `failbit`, `badbit`, `eofbit`.
Возьмите те, что заставляют `ifstream::operator bool()` возвращать `false`, то есть заставляют конструкцию вроде `std::cin >> x` вернуть `false`.

Запускайте решение с одним параметром командной строки: `.in`-файл теста.
Проверьте, что вывод совпал с `.out`-файлом.

Отдельно попробуйте обработать вообще все ошибки через исключения и посмотрите, какие тесты пройдут, а какие упадут.
**ВАЖНО**: убедитесь, что `.in`-файлы не поменялись, важен каждый байт, особенно в конце файла.

_Примечание_: здесь для демонстрации важны пробельные символы, поэтому упражнение специально читает из файла, а не с экрана.
Так сложнее потерять пробельные символы при вводе.

Теперь вы умеете не писать `if (std::cin)` после каждой операции чтения.

# Для домашки (`1x`)
## `10-size-mismatch-checker`
Скопируйте свой `vector_size_mismatch` из предыдущего задания и добавьте удобства использования и инкапсуляции:

1. Добавьте вспомогательную функцию `ensure_same_size`, которая проверяет, что два `vector<int>` одного размера, и кидает исключение, если это не так.
   Теперь можно не писать `if`, а вызывать эту функцию; в два раза меньше кода и возможностей опечататься.
2. Сделайте все конструкторы `vector_size_mismatch` приватными, чтобы заставить всех вызывать `ensure_same_size`.
3. (сложная часть, можно пропустить) Сделайте так, чтобы `ensure_same_size` работала с любыми векторами.

Теперь вы умеете писать функции-проверяторы.

# Продвинутый синтаксис (`2x`)
## `20-mixed-exceptions`
Не запуская код, догадайтесь, какой будет вывод.
Объясните происходящее.
Объясните, почему это не UB.

Теперь вы знаете, что исключения могут путаться.

## `21-unexpected-exception`
Программа считывает в цикле два числа, выводит ошибку, если они не равны, выводит число, если равны.
К сожалению, в ней баг, и ошибка странная: не содержит ни `Expected equal numbers`, ни два числа.
Сначала исправьте баг, а затем исправьте `catch` так, чтобы при допущении такого бага программа очевидно не работала (может компилироваться).

Теперь вы ещё лучше знаете, что исключения могут путаться.

## `22-injection-attack`
Не меняя код программы добейтесь того, чтобы она вывела в точности содержимое файла `22-injection-attack.sol`.
Вы можете дать на вход любой файл.

Теперь вы знаете, что не надо просто так передавать куда-то строчки, которые ввёл пользователь.
