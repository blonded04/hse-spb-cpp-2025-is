# Общее
Цель занятия: порешать упражнения на стандартную библиотеку языка C++, а точнее — библиотек
[контейнеров](https://en.cppreference.com/w/cpp/container),
[итераторов](https://en.cppreference.com/w/cpp/iterator) и
[алгоритмов](https://en.cppreference.com/w/cpp/algorithm).

В каждом задании вы узнаете новый кусочек теории.

Оставшиеся задания выполните дома, хотя это и не даёт никаких баллов :(.

# Базовые упражнения
## `01-to-string`
Реализуйте функцию `to_string` для вектора целых чисел, которая работает с итераторами.

## `02-adjust`
Почитайте документацию, разберитесь как смещать итератор влево и вправо, причем сразу на несколько элементов.

## `03-to-string-list`
В этом задании реализация функции `to_string` должна в точности совпадать с решением задания `01-to-string`,
несмотря на то, что использует `std::list` вместо `std::vector`.

## `04-to-string-const`
В этом задании потребуется `const_iterator` — специальный итератор,
через который нельзя менять содержимое контейнера.
Вам потребуется чуть-чуть поменять сигнатуру (типы аргументов, возвращаемое значение) функции `to_string`,
а тело останется неизменным.

## `14-map-lower-bound`
Обратите внимание, как `std::lower_bound` при наивном использовании на `map` работает за линейное время:
он не может обратиться к элементу по номеру и не может воспользоваться внутренней структурой `map`.
Остаётся только перебрать все элементы.

## `22-erase-algo`
Обратите внимание, что ни один из алгоритмов технически не может поменять контейнер,
с которым работает: ему на вход подаются только итераторы.

## `30-invalidation`
В этом задании потребуется придумать, что можно сделать с `vector`'ом,
чтобы ссылка на его первый элемент перестала быть корректной (инвалидировалась)
и при обращении по ней произошло UB.
Например, это происходит при _реаллокации_: если перестало хватать места в памяти,
`vector` переезжает в другое место.
