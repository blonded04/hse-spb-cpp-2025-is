# Общее

Цель занятия: поупражняться с шаблонами.

Изменять можно только строчки между `TODO START`/`TODO END`, а также помеченные `TODO`.

Не забывайте про doctest (`doctest_main.cpp` в частности).

# Базовые упражнения (`0x`)
## `00-generic-function`
Упражнение на шаблоны: функция `measure_call` принимает функциональный объект, с типом возвращаемого значения `void` при вызове от нуля аргументов, и выводит в `std::cout` время, потраченное на исполнение кода этого функционального объекта.
На данный момент эта функция принимает функциональный объект как `std::function<void()>`. Не меняя поведения программы, измените сигнатуру функции `measure_call`, поменяв только тип аргумента функции, избавившись от `std::function<void()>`.

Избавление от `std::function` может быть полезно, чтобы, например, [не делать лишнюю аллокацию](https://stackoverflow.com/q/46163607/9774145), когда можно этого избежать.

## `01-stack`
Это упражнение на шаблоны классов. Напишите шаблон класса `stack<T>`, который поддерживает операции `push`/`pop`, а внутри использует `std::vector<T>` для хранения данных.

Реализацию напишите целиком в `stack.hpp` — это стандартная конвенция. Так как шаблон нельзя скомпилировать заранее, не зная точный тип, определения (почти) всех шаблонов пишутся в заголовках. На ODR это не влияет.

## `OPTIONAL_TEST_INIT`
Здесь вам достаточно сделать `optional<>`, который хранит внутри себя
булевский флаг, правильно выровненный массив из `unsigned char` для объекта,
и корректно его инициализирует в конструкторах.

Полноценное правило трёх пока не требуется.

## `OPTIONAL_TEST_OWNERSHIP`
Здесь потребуется реализовать деструктор.

## `OPTIONAL_TEST_RESET`
Сделайте метод `reset()`, который обнуляет `optional<>`.
Практически как деструктор.
Возможно, можно что-то выразить через другое?

## `OPTIONAL_TEST_COPYABLE`
Добавьте copy-конструктор и оператор присваивания.
Код разрастается.

## `02-full-class-specialization`
Класс `call_result_storage<T>` позволяет вызывать произвольный функтор, возвращающий `T`, методом `call`.
Он сохраняет результат вызова в поле.
К сожалению, при `T=void` всё ломается.
Почините.

## `03-partial-class-specialization`
Класс `caller` сохраняет внутри себя произвольный функтор и может его вызвать при помощи `operator()`.
Поддержите сохранение вектора функторов: `operator()` пусть вызывает каждый функтор по очереди.
