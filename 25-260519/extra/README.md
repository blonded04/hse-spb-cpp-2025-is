Разрешается изменять только код, помеченный `TODO`.

При компиляции сетевых приложений под Windows через gcc/clang может потребоваться прилинковать библиотеку `ws2_32` (ключ `-lws2_32`).
В CMake:
```
if (WIN32 AND NOT MSVC AND NOT BORLAND)
    target_link_libraries(main ws2_32)
endif ()
```

При компиляции многопоточных приложений под Linux/macOS может потребоваться добавить ключ `-pthread` для компиляции и линковки (не `-lpthread`, это [прям отдельный ключ компилятора](https://stackoverflow.com/questions/23250863/difference-between-pthread-and-lpthread-while-compiling/23251828#23251828)).
В CMake:
```
find_package(Threads REQUIRED)
target_link_libraries(main PRIVATE Threads::Threads)
```

# Синтаксис (`0x`)
## `01-netcat-client`, `02-netcat-server`, `03-netcat-client-server`
Проверьте, что вы можете локально при помощи утилиты Netcat или похожей:

1. Создавать TCP-соединение к произвольному компьютеру на произвольный порт, то есть быть TCP-клиентом.
2. Принимать TCP-соединения на произвольный порт, то есть быть TCP-сервером.
3. Объединять предыдущие два пункта: подключаться своим клиентом к своему серверу.

Возможные проблемы:

* Вы пытаетесь создать сервер на порте с номером < 1024, под Linux это может быть доступно только `root`.
  Слишком важные порты.
* У вас включён SELinux/антивирус/файрволл, который запрещает "необычные" сетевые соединения.
  Обычно запрещают создавать сервера на произвольных портах.
* Иногда могут разрешать подключения только с конкретных IP-адресов.
  Например, только с компьютера на себя же, но не с других компьютеров.
* Вы создали сервер на IPv4, а подключаетесь по IPv6.
  Например, имя `localhost` в зависимости от контекста и настроек может превращаться как в IPv4 (`127.0.0.1`),
  так и в его IPv6 аналог (`::1`).
* Конкретно ваш интернет-провайдер перехватывает нешифрованные HTTP-запросы.
  Например, особенно так любят делать мобильные провайдеры, но не только ([1](https://habr.com/ru/post/506218/), [2](https://habr.com/ru/post/489528/), [3](https://habr.com/ru/post/493402/)).
  Весной 2022 года такие проблемы были с Тинькофф Мобайл.

# Хитрые упражнения (`3x`)
## `30-messages-protocol`
Протокол TCP передаёт 'байты', а не сообщения.
Но иногда хочется передать именно несколько сообщений разной длины.
Даже если передавать их "за одну команду", то TCP никаких границ между командами не запоминает.
Придёт тот же набор байт в таком же порядке, но, возможно, по-другому порезаный.

Придумайте какой-нибудь протокол, который позволит сделать две функции: `send_message` отправляет произвольное сообщение в сокет,
парная ей `read_message` читает произвольное сообщение.
Принципиально есть два варианта:

1. После каждого сообщения ставить некоторый разделитель. Но тогда надо, чтобы этот разделитель не встречался в самом сообщении.
2. Перед каждым сообщением запистаь его длину в байтах в каком-то формате.
